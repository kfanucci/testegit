Códigos para o terminal git

$ git init

    inicia o git na pasta local

  • --bare (repositório para servidor, para armazenamento)

$ git add "nome arquivo"

    adiciona o arquivo no git

    com . => adc todas pastas do repositório

    sempre que for commitar tem que add

$ git rm "nome arquivo"

    apaga arquivo

$ git commit -m "mensagem"

    Usado para comitar alguma alteração no arquivo

    Não comitar sem fechar o código

$ git status

    mostra o status do git o que precisa ser feito

$ git config

  • --local user.name "kauan"
  • -- global user.email "kauan@kauan.com

.gitignore (cria um arquivo)

    todo arquivo dentro do repositorio dentro do arquivo .gitignore sera ignorado

$ git log

    ver histórico de commits

  • --oneline (ver tudo em uma linha
  • -p (ver commit inteiro com as alterações)
  • --pretty (estiliza o formato)

$ git remote (mostra os locais remotos)

  • add "origin" "url pasta ou link"
  • -v (mostra os caminho do remote)

$ git clone "url do servidor" "nome da pasta nova"

    usar no computador de outra pessoa

$ git branch (ramo)

    mostra os branchs atuais

  • "nome" cria uma nova ramificação - linha do tempo

$ git checkout (usado para navegar pelo git)

  • "nome branch" muda o git para a branch
  • --"nome do arquivo" desfaz as alteração antes do commit
  • "numero hash" vai para o hash

$ git merge

  • "nome branch" mescla 2 linhas do tempo porém tem que estar na master e todos commit vão para master

$ git release

  • "nome branch" atualiza a branch atual

$ git pull "origin" "master"

    puxa do local remoto para a branch que quiser

$ git push "origin" "mater"

    empurra os dados p/ o local remoto na branch

$ git reset HEAD "nomeArquivo"

    head = estado que o trabalho esta

    volta ao estado para commitar

$ git revert "codigo hash"

    pegando o hash do commit pelo log, desfaz o commit

    pode usar os 7 primeiros numeros do hash

$ git stash

    salva uma alteração para depois

  • list (mostra uma lista dos stash
  • pop (adiciona para commit e tira do stash

$ git diff "hash0".."hash1"

    mostra diferença de um hash para outro

    apenas diff mostra tudo antes do commit

$ git tag -a "v.1.0' -m "mensagem"

    cria um marco no projeto, adiciona uma versão

  • git push origin v.1.0 (manda versão pro remoto)

exemplo de funcionalidade
